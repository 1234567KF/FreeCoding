# Annotation Templates — 注释模板与示例

> 本文件为 kf-ui-prototype-generator 的注释标准模板库，包含所有层级的填写模板、完整业务示例、及详略对比参考。

---

## 第一部分：完整注释模板

> 以下模板用于嵌入 HTML 原型的暗门注释抽屉（Ctrl+B 可见），每个层级对应一个 Tab。

---

### L0 — 页面概览 <!-- 必填 -->

**用途**：定义页面身份、所属模块、业务上下文，使读者一目了然地理解页面定位。

```html
<!-- L0: 页面概览 -->
<div class="anno-tab-content" id="anno-tab-l0">
  <h3>页面概览</h3>
  <table>
    <tr><th>属性</th><th>内容</th></tr>
    <tr><td>页面名称</td><td><!-- 填写页面标题 --></td></tr>
    <tr><td>所属模块</td><td><!-- 填写一级/二级模块路径 --></td></tr>
    <tr><td>页面类型</td><td><!-- List / Form / Detail / Dashboard / Composite --></td></tr>
    <tr><td>业务说明</td><td><!-- 一句话描述该页面的核心业务价值 --></td></tr>
    <tr><td>目标用户</td><td><!-- 使用该页面的角色列表 --></td></tr>
    <tr><td>PRD 来源</td><td><!-- [PRD x.x.x] 章节引用 --></td></tr>
  </table>
</div>
```

```html
  </tr>
  <tr>
    <td><!-- 字段名或操作名 --></td>

    <td><!-- RW / R / H / — --></td>
    <td><!-- RW / R / H / — --></td>
  </tr>

```

## 第二部分：典型业务页面完整注释示例

> 以「员工管理列表页」为例，演示所有层级的实际填充效果。

---

### L0 — 页面概览

| 属性 | 内容 |
|------|------|
| 页面名称 | 员工管理列表 |
| 所属模块 | 人事管理 > 员工管理 |
| 页面类型 | List |
| 业务说明 | 企业员工信息的集中管理入口，支持多条件查询、批量操作及数据导出，涵盖员工全生命周期管理 |
| 目标用户 | Admin（系统管理员）、Manager（部门经理）、User（HR专员） |
| PRD 来源 | [PRD 3.2.1] 员工管理模块 |

---

### L0.ops — 操作定义

| 操作 | 说明 | 可执行角色 | 备注 |
|------|------|-----------|------|
| 查询 | 按条件筛选员工列表 | Admin, Manager, User | Manager 仅可查询本部门 |
| 新增 | 录入新员工信息 | Admin, User | 触发方式：顶部「新增员工」按钮 |
| 编辑 | 修改现有员工信息 | Admin, User | 操作列「编辑」按钮；离职状态不可编辑 |
| 删除 | 逻辑删除员工记录 | Admin | 操作列「删除」按钮；需二次确认 |
| 查看详情 | 进入员工详情页 | Admin, Manager, User | 点击姓名链接或操作列「查看」 |
| 批量导入 | Excel 批量导入员工 | Admin | 顶部「批量导入」按钮；需下载模板 |
| 批量删除 | 勾选后批量逻辑删除 | Admin | 需至少选中1条；二次确认 |
| 导出 | 导出当前筛选结果为 Excel | Admin, Manager | 顶部「导出」按钮；最大导出10000条 |
| 状态变更 | 转正/离职/复职操作 | Admin, User | 操作列下拉菜单；触发状态机流转 |

### L0.deps — 业务关联

| 关联模块 | 关联类型 | 关联说明 | 级联策略 |
|---------|---------|---------|---------|
| 部门管理 | 数据源 | 员工所属部门下拉数据来源；部门变动影响员工归属 | 级联更新：部门名称变更同步显示 |
| 薪资系统 | 消费者 | 员工基本信息作为薪资计算基础；离职触发薪资结算 | 阻断：有未结算薪资时不允许删除 |
| 考勤系统 | 消费者 | 员工信息作为考勤归属依据 | 级联更新：部门变更同步考勤分组 |
| 权限系统 | 双向 | 员工角色决定系统权限；离职自动回收权限 | 级联删除：离职时清除所有系统角色 |
| 合同管理 | 触发者 | 员工转正/离职触发合同状态变更 | 级联更新：状态变更通知合同模块 |

### L2.exceptions — 异常处理

| 异常场景 | 触发条件 | 系统响应 | 用户提示 | 恢复方案 |

### L3 — 状态机

**实体**：员工（Employee）

**状态定义**：
- 试用期（Probation）— Tag: 蓝色
- 在职（Active）— Tag: 绿色
- 离职（Resigned）— Tag: 灰色

**状态转移表**：

| 当前状态 | 触发事件 | 目标状态 | 权限要求 | 前端展示 |
|---------|---------|---------|---------|---------|
| 试用期 | 转正操作 | 在职 | Admin, User(HR) | 操作列出现「转正」按钮；转正后 Tag 变绿 |
| 试用期 | 离职操作 | 离职 | Admin, User(HR) | 操作列出现「离职」按钮；需填写离职原因 |
| 在职 | 离职操作 | 离职 | Admin, User(HR) | 操作列出现「离职」按钮；触发薪资结算流程 |
| 离职 | 复职操作 | 试用期 | Admin | 操作列出现「复职」按钮；重新分配部门和岗位 |

**不可达转移**：
- 在职 → 试用期（禁止降级）
- 离职 → 在职（必须经过试用期）

**状态对操作的影响**：

| 状态 | 可用操作 | 不可用操作 |
|------|---------|-----------|
| 试用期 | 查看、编辑、转正、离职、导出 | 删除 |
| 在职 | 查看、编辑、离职、导出 | 删除、转正 |
| 离职 | 查看、复职(Admin)、导出 | 编辑、删除、转正 |

### 详略对比总结

| 对比维度 | 简单 CRUD（系统配置） | 复杂业务（订单审批） |
|---------|---------------------|---------------------|
| 涉及层级 | L0 + L1 + L1.bounds + L1.constraints + L2（极简）+ L6 | **全部层级**均有实质内容 |
| L0.ops 行数 | 2行（查看+保存） | 7行（含批量、转审等） |
| L0.deps | 省略（无外部依赖） | 5个关联模块 |
| L1.perm | 省略（单角色） | 3角色完整矩阵 |
| L1.constraints | 2条（单记录+生效策略） | 4条以上（阈值+条件+互斥） |
| L2 规则数 | 2条 | 5+条核心规则 |
| L2.exceptions | 省略（无特殊异常） | 3+条异常场景 |
| L3 状态机 | 省略（无状态流转） | 6+条状态转移 |
| 总注释量 | ~40行 | ~150+行 |

**原则**：注释量 = f(业务复杂度)。CRUD 页面只需确保字段约束清晰；复杂业务页面须 MECE 覆盖所有规则、异常和状态流转，确保 AI 和人类均无盲区。
