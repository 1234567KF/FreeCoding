# Spec 前置准备质量评分标准

> 本文档定义 kf-spec Step 1 需求澄清阶段的质量评分体系，用于在进入 Step 2 生成 Spec 草稿前，量化评估前置准备的充分程度。

## 一、评分维度

采用 4 维度评分，每维度 1-5 分，总分 20 分。

### D1 — 需求定义清晰度

评估 Step 1 四个澄清问题（Q1-Q4）的回答质量。

| 分数 | 标准 |
|------|------|
| 5 | 4 个澄清问题都有具体、无二义性的答案，目标用户可精确到角色名称 |
| 4 | 4 个问题都有答案，但 1 个回答略有模糊（如用户角色仅说"管理员"未区分超管/普通管理员） |
| 3 | 2-3 个问题有清楚答案，1 个存疑或过于笼统 |
| 2 | 仅 1-2 个问题有明确答案，核心问题（Q2 或 Q3）回答不清 |
| 1 | 多数问题无明确答案，需求处于"想法阶段" |

### D2 — 用户场景覆盖度

评估需求描述中用户使用场景的覆盖程度。

| 分数 | 标准 |
|------|------|
| 5 | 核心场景 ≥2 个，每个场景包含触发条件、操作步骤、预期结果，失败路径已初步考虑 |
| 4 | 核心场景 ≥2 个，操作流程清晰，但失败路径仅部分覆盖 |
| 3 | 核心场景 1 个且描述完整，或 ≥2 个但描述较粗 |
| 2 | 仅描述了大致使用方式，未形成完整场景 |
| 1 | 场景模糊，仅有功能点列表，无用户操作流程 |

### D3 — 技术可行性评估

评估用户对技术风险和约束的了解程度。

| 分数 | 标准 |
|------|------|
| 5 | 已评估主要技术风险（≥2 项），每项有应对方案，技术选型已初步确定 |
| 4 | 已识别主要技术风险，应对方案部分确定 |
| 3 | 知道存在技术风险但方案未定，或技术选型有待确认 |
| 2 | 仅模糊提及"可能有风险"，未具体分析 |
| 1 | 未评估技术可行性，或对技术约束完全不了解 |

### D4 — 规范完备性

评估输入输出格式、性能目标等规范的定义程度。

| 分数 | 标准 |
|------|------|
| 5 | 输入输出格式初步定义（至少列出核心字段），性能目标有初步数字（如"响应时间 <500ms"） |
| 4 | 输入输出有描述但未精确到字段级别，性能有定性要求（如"秒级响应"） |
| 3 | 部分输入输出已定义，性能未提及但场景复杂度较低 |
| 2 | 输入输出仅有笼统描述，无格式定义 |
| 1 | 缺少任何规范性定义 |

## 二、综合评分与判定

**计算方式**：均分 = (D1 + D2 + D3 + D4) / 4

| 评级 | 均分范围 | 总分范围 | 判定结果 |
|------|---------|---------|---------|
| 优秀 | ≥4.0 | ≥16 | 直接进入 Step 2，前置准备充分 |
| 良好 | 3.0-3.9 | 12-15 | 进入 Step 2，但标记薄弱维度（得分 ≤3 的维度）供后续重点关注 |
| 不足 | <3.0 | <12 | 返回 Step 1 补充澄清，提示需要加强的具体维度 |

**"良好"级别的补充说明**：

当均分为 3.0-3.9 时，AI 在进入 Step 2 前应向用户展示薄弱维度提示：

```
⚠️ 前置准备质量评分：均分 X.X（良好）
以下维度得分较低，Spec 生成时将重点关注：
- [维度名称]（X 分）：建议补充 [具体建议]
```

## 三、评分操作流程

**执行时机**：Gate 1 通过后、进入 Step 2 前（即 Gate 1.5）。

**操作步骤**：

1. 回顾 Q1-Q4 的回答内容
2. 按 4 个维度逐一评分（1-5 分）
3. 计算均分并确定评级
4. 向用户展示评分结果表格：

```
### Gate 1.5 — 前置准备质量评分

| 维度 | 得分 | 说明 |
|------|------|------|
| D1 需求定义清晰度 | X/5 | [评分依据简述] |
| D2 用户场景覆盖度 | X/5 | [评分依据简述] |
| D3 技术可行性评估 | X/5 | [评分依据简述] |
| D4 规范完备性 | X/5 | [评分依据简述] |
| **均分** | **X.X** | **[评级：优秀/良好/不足]** |
```

5. 根据评级决定后续动作：
   - 优秀/良好：继续进入 Step 2
   - 不足：列出需要补充的维度和具体建议，返回 Step 1

## 四、与复杂度分级的联动

评分完成后，同步判定需求复杂度等级（参见 [complexity-based-prep-guide.md](complexity-based-prep-guide.md)）：

- **L1 小微需求**（预估 ≤4h）：评分目标 均分 ≥3.5
- **L2 中等需求**（预估 4-16h）：评分目标 均分 ≥4.0
- **L3 复杂需求**（预估 ≥2 天）：评分目标 均分 ≥4.5

当实际评分低于对应复杂度的目标分时，向用户发出提示：

```
⚠️ 当前需求判定为 L[X] 级别，建议前置准备评分均分 ≥X.X，当前为 X.X。
建议在进入 Step 2 前补充以下准备工作：[对应级别的额外准备物]
```
