# 任务拆解示例：用户注册模块

> 本文档展示如何将一个完整需求拆解为可独立执行的编码任务。以"用户注册模块"为场景演示。

---

## 需求概述

实现用户注册功能，支持手机号+验证码注册，包含参数校验、验证码验证、用户创建、欢迎通知四个核心环节。

---

## 任务依赖关系图

```
T001 [数据库表结构] ──┬──→ T003 [注册接口]
T002 [验证码接口]   ──┘        │
                              ├──→ T005 [欢迎通知]
T004 [前端注册页面] ───────────┘        │
                                       ↓
T006 [接口测试] ←── T003, T005
T007 [端到端测试] ←── T004, T006
```

**可并行组**：T001 + T002（无依赖）、T004（仅依赖接口契约，可与后端并行）

---

## 任务卡片列表

### T001 — 创建用户表结构

| 字段 | 内容 |
|------|------|
| **任务 ID** | T001 |
| **标题** | 创建用户表结构与 Entity 定义 |
| **描述** | 创建 `t_user` 表的数据库迁移脚本和对应的 Entity 类，包含手机号、密码哈希、昵称、注册时间等字段 |
| **输入** | PRD 第 3 节数据字段定义 |
| **输出** | `db/migrations/V001__create_user_table.sql`、`src/entity/user.rs` |
| **依赖任务** | 无 |
| **验收标准** | ① 迁移脚本可执行无报错 ② Entity 字段与 PRD 字段定义一一对应 ③ 手机号字段有唯一索引 |
| **预估时间** | 40 分钟 |

### T002 — 实现验证码发送接口

| 字段 | 内容 |
|------|------|
| **任务 ID** | T002 |
| **标题** | 实现短信验证码发送接口 |
| **描述** | 实现 `POST /api/v1/sms/send-code` 接口，生成 6 位随机验证码，存储到缓存（5 分钟过期），调用短信服务发送 |
| **输入** | PRD 第 2 节业务规则 R003（验证码规则） |
| **输出** | `src/controller/sms_controller.rs`、`src/service/sms_service.rs` |
| **依赖任务** | 无 |
| **验收标准** | ① 同一手机号 60 秒内不可重复发送 ② 验证码 5 分钟后自动过期 ③ 返回标准响应格式 |
| **预估时间** | 1.5 小时 |

### T003 — 实现用户注册接口

| 字段 | 内容 |
|------|------|
| **任务 ID** | T003 |
| **标题** | 实现用户注册核心接口 |
| **描述** | 实现 `POST /api/v1/user/register` 接口，校验手机号格式与验证码，创建用户记录，返回注册成功信息 |
| **输入** | PRD 第 2 节业务规则 R001-R005、T001 输出的 Entity |
| **输出** | `src/controller/user_controller.rs`、`src/service/user_service.rs` |
| **依赖任务** | T001（用户表结构）、T002（验证码验证逻辑） |
| **验收标准** | ① 手机号格式校验通过 ② 验证码错误返回 400 ③ 手机号已注册返回 409 ④ 密码以哈希形式存储 ⑤ 注册成功返回用户 ID |
| **预估时间** | 2 小时 |

### T004 — 实现前端注册页面

| 字段 | 内容 |
|------|------|
| **任务 ID** | T004 |
| **标题** | 实现前端注册页面与表单交互 |
| **描述** | 按 PRD 第 4 节交互规范，实现注册表单页面，包含手机号输入、获取验证码按钮（60 秒倒计时）、密码输入、提交按钮 |
| **输入** | PRD 第 4 节页面交互逻辑、接口契约文档 |
| **输出** | `src/pages/register/index.tsx`、`src/pages/register/register.module.css` |
| **依赖任务** | 无（基于接口契约并行开发） |
| **验收标准** | ① 表单校验提示正确 ② 获取验证码后 60 秒倒计时 ③ 加载/空/错误三种状态正确展示 ④ 注册成功跳转至首页 |
| **预估时间** | 2 小时 |

### T005 — 实现注册欢迎通知

| 字段 | 内容 |
|------|------|
| **任务 ID** | T005 |
| **标题** | 实现注册成功后的欢迎通知 |
| **描述** | 注册成功后异步发送欢迎短信，记录发送日志。发送失败不影响注册主流程 |
| **输入** | PRD 第 2 节业务规则 R006（通知规则） |
| **输出** | `src/service/notification_service.rs`、`src/event/user_registered_handler.rs` |
| **依赖任务** | T003（注册接口触发事件） |
| **验收标准** | ① 注册成功后 5 秒内触发通知 ② 通知失败不阻塞注册响应 ③ 发送记录写入日志表 |
| **预估时间** | 1 小时 |

### T006 — 编写接口自动化测试

| 字段 | 内容 |
|------|------|
| **任务 ID** | T006 |
| **标题** | 编写注册模块接口测试用例 |
| **描述** | 针对验证码发送和用户注册两个接口，编写覆盖正常流程、边界条件、异常场景的接口测试 |
| **输入** | T002、T003 的接口实现、PRD 验收标准 |
| **输出** | `tests/api/test_register.rs` |
| **依赖任务** | T003（接口实现完成） |
| **验收标准** | ① 正常注册流程测试通过 ② 重复手机号测试返回 409 ③ 错误验证码测试返回 400 ④ 测试覆盖率 ≥ 80% |
| **预估时间** | 1.5 小时 |

### T007 — 编写端到端测试

| 字段 | 内容 |
|------|------|
| **任务 ID** | T007 |
| **标题** | 编写注册流程端到端测试 |
| **描述** | 模拟用户从打开注册页面到注册成功的完整流程，验证前后端协作正确性 |
| **输入** | T004 前端页面、T006 接口测试通过 |
| **输出** | `tests/e2e/test_register_flow.rs` |
| **依赖任务** | T004（前端页面）、T006（接口测试通过） |
| **验收标准** | ① 完整注册流程可走通 ② 错误提示在页面正确展示 ③ 注册成功后正确跳转 |
| **预估时间** | 1 小时 |

---

## 执行计划总览

| 阶段 | 并行任务 | 预估耗时 |
|------|---------|---------|
| 第 1 阶段 | T001 + T002 + T004 | 2 小时 |
| 第 2 阶段 | T003 | 2 小时 |
| 第 3 阶段 | T005 + T006 | 1.5 小时 |
| 第 4 阶段 | T007 | 1 小时 |
| **总计** | **7 个任务** | **约 6.5 小时**（并行后） |
