# SDD Excel 模板映射指南

> 适用范围：任何使用 SDD 需求采集 Excel 模板（v2.2+）的项目。
> 本指南供 `kf-spec` 和 `prd-generator` Skill 在解析 SDD Excel 时参考。

---

## 一、模板概述

SDD（Software Design Document）需求采集 Excel 是一套标准化的 18-Sheet 需求采集模板，通过结构化表格统一收集项目基础信息、业务流、数据模型、功能页面、权限、状态机、业务规则、接口契约、非功能需求等关键设计要素。

**识别特征**：
- 文件名通常包含 `SDD需求采集`
- 必含 `ReadMe` Sheet 和 `Sheet0-AI指令配置` / `Sheet1-项目基础信息`
- Sheet 命名格式：`SheetN-中文名称`（N 为 0~18）

## 二、AI 读取通用规则

| 规则类型 | 解析策略 | 核心要点 |
|---------|---------|---------|
| 合并单元格 | 内容仅在左上角格子，其他格子为空。遇到空单元格时向左/向上回溯获取分类 | 只读左上角，空值回溯填充 |
| Checkbox 多选项 | `☑` = 推荐默认（已勾选）= 优先采用 / 必选；`☐` = 可选未默认 = 可选项 | 将 `☑`/`☐` 转为结构化布尔值，记录选用状态 |
| 单元格内换行 | 一个格子里用 `\n` 分隔多行文本，每行是一个独立选项 | 按 `\n` 逐行解析 |
| 颜色含义 | 浅绿=示例值；浅黄=可选项区；白色=数据填写区 | 颜色仅辅助参考，不改变内容语义 |
| 空行跳过 | 部分行可能全为空，是结构占位 | 跳过全空行 |
| Sheet 依赖 | Sheet9 依赖 Sheet3 业务流序号；Sheet6 依赖 Sheet5 页面名称 | 按依赖顺序解析，后读 Sheet 引用先读 Sheet |

## 七、Sheet 间依赖关系

```
Sheet5(功能页面表) ──→ Sheet6(功能权限表)   [页面名称引用]
Sheet5(功能页面表) ──→ Sheet9(页面交互)     [页面序号引用]
Sheet5(功能页面表) ──→ Sheet11(接口契约)    [页面序号引用]
Sheet2(业务流) ──────→ Sheet8(业务规则)     [步骤序号引用]
Sheet3(数据流) ──────→ Sheet4(数据关系)     [实体名称引用]
Sheet7(状态流转) ────→ Sheet8(业务规则)     [状态实体引用]
```

**解析顺序要求**：被引用的 Sheet 必须先于引用它的 Sheet 解析。

## 九、异常处理

| 场景 | 处理方式 |
|------|---------|
| Sheet 不存在或为空 | 跳过该 Sheet，在对应章节标注「待补充」 |
| 必填列缺失 | 记录为 `[TODO: 待讨论]`，不自行补全 |
| 引用关系断裂（如 Sheet9 引用了 Sheet5 不存在的页面编号） | 标注异常，要求人工核实 |
| Checkbox 解析歧义 | 以「选用值」列为准，该列为空则以 `☑` 项为准 |
| 单元格合并导致数据错位 | 按 ReadMe 规则向左/向上回溯填充 |
