# 迁移指南 — 从 Claude Code 到通用 IDE

## 概述

本指南帮助已从 Claude Code 使用 ChronoCoding 的用户迁移到 Qoder、Trae、Cursor 等通用 IDE。

## 主要变化

### 1. Agent 并发 → 串行执行

| 特性 | Claude Code | 通用 IDE |
|------|-------------|----------|
| 执行模式 | Agent() spawn 真并发 | 当前会话按角色切换串行执行 |
| 三队执行 | 红蓝绿同时运行 | 红→蓝→绿顺序执行 |
| 产物隔离 | 进程隔离 | 文件隔离（red-*.md / blue-*.md / green-*.md） |
| 状态追踪 | ruflo swarm 面板 | hammer-bridge.cjs + hang-state.json |

### 2. 自动 Hook → 手动触发

| Hook | Claude Code | 通用 IDE |
|------|-------------|----------|
| PreToolUse | 自动捕获 | 手动调用 token-tracker.cjs |
| PostEdit | 自动触发 | 手动调用 alignment-hook.cjs |
| 生命周期 | 自动执行 | 手动执行 hook-handler.cjs |

### 3. AskUserQuestion → 文件交互

| 场景 | Claude Code | 通用 IDE |
|------|-------------|----------|
| 问卷弹窗 | AskUserQuestion API | 输出到文件 + 用户回复后提交 |
| 门控阻断 | 弹窗等待 | 状态机暂停 + 文件系统等待 |

### 4. MCP 依赖 → 独立进程

| MCP | Claude Code | 通用 IDE |
|-----|-------------|----------|
| ruflo | `claude mcp add ruflo` | 不使用（状态机替代） |
| nano-banana | `claude mcp add nano-banana` | 直接调用 Gemini API |
| lean-ctx | MCP 工具 | 命令行工具 |

## 迁移步骤

### 步骤 1: 备份原配置

```bash
cp -r .claude .claude-backup
```

### 步骤 2: 运行安装脚本

```powershell
# Windows
.\install.ps1 -IdeName qoder

# macOS/Linux
./install.sh --ide qoder
```

### 步骤 3: 迁移自定义技能

将 `.claude/skills/` 下的自定义技能复制到 `{IDE_ROOT}/skills/`，并替换路径：

```bash
# 批量替换 .claude/ → {IDE_ROOT}/
find {IDE_ROOT}/skills -type f \( -name "*.md" -o -name "*.cjs" -o -name "*.json" \) -exec sed -i 's/\.claude\//{IDE_ROOT}\//g' {} +
```

### 步骤 4: 配置模型

编辑 `model-config.json`，填入你的 API Key。

### 步骤 5: 验证

```bash
# 测试 /夯 流程
node {IDE_ROOT}/helpers/hang-state-manager.cjs --init "测试任务" --depth A
node {IDE_ROOT}/helpers/hang-state-manager.cjs --dashboard
```

## 注意事项

1. **缓存优化**：串行模式下共享前缀缓存效果反而更好（顺序请求命中率高）
2. **Token 消耗**：串行执行总 Token 与并发相当，但首响应时间更长
3. **中断恢复**：hang-state.json 支持断点续传，比 Claude Code 更可靠
4. **模型切换**：通用 IDE 无自动模型路由，需在技能 frontmatter 声明或手动切换

## 故障排除

| 问题 | 解决 |
|------|------|
| 技能未触发 | 检查触发词是否在 IDE 配置中注册 |
| 路径错误 | 确认 `{IDE_ROOT}` 已替换为实际目录 |
| API Key 无效 | 检查 `model-config.json` 格式 |
| 状态丢失 | 确认 `.claude-flow/` 目录有写入权限 |
